# WearBrowser Documentation

WearBrowser 的 `docs/` 是项目的设计中心，记录产品理念、架构决策、交互规范、功能说明、工程标准与未来路线。

## 文档目录

- `01-product/`：产品定位、愿景、路线、功能总览。
- `02-architecture/`：架构、核心引擎、性能、Feature Flags。
- `03-ui/`：设计系统、圆屏适配、OLED、手势、沉浸模式。
- `04-features/`：缩放、Reader、收藏、历史、下载、站点 Profile 等功能设计。
- `05-development/`：贡献、代码规范、测试、Benchmark、发布、CI/CD。
- `06-future/`：v2 插件系统、同步能力、实验方向。

## 项目信条

1. Watch First，而不是 Phone Mini。
2. Gesture First，而不是 Button First。
3. Content First，而不是 Browser Chrome First。
4. Performance is a Feature。
5. Every Pixel Matters。
6. Minimal UI, Maximum Content。
7. Architecture Before Features。
8. Consistency Over Cleverness。
9. Privacy by Default。
10. Open, Composable, Replaceable。

任何重要功能进入开发前，都应先在 `docs/` 中完成设计说明。
